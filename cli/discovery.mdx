## Overview

The `discovery` command group provides powerful automated tools for discovering new creators across multiple platforms. It uses trending topics, category exploration, and intelligent evaluation to find high-quality creators that match your criteria.

## Architecture

The discovery system consists of several components:

- **Discovery Pipeline** - Orchestrates the entire discovery workflow
- **Trending Discovery** - Finds creators based on trending topics and hashtags
- **Creator Evaluator** - Scores creators based on quality metrics
- **Duplicate Detector** - Prevents duplicate entries
- **Queue System** - Manages discovery jobs with BullMQ

## Commands

### Start Discovery Pipeline

Start the automated discovery pipeline:

```bash
tiktok-miner discovery start [options]
```

#### Options

- `--mode <mode>` - Discovery mode (trending, category, both) (default: both)
- `--platforms <platforms>` - Comma-separated platform list (default: all)
- `--interval <hours>` - Hours between discovery runs (default: 24)
- `--quality-threshold <score>` - Minimum quality score (0-100) (default: 60)
- `--daemon` - Run as background daemon

#### Examples

**Start trending discovery for all platforms:**

```bash
tiktok-miner discovery start --mode trending
```

**Run as daemon with custom threshold:**

```bash
tiktok-miner discovery start --daemon --quality-threshold 70 --interval 12
```

**Discover only from Instagram and TikTok:**

```bash
tiktok-miner discovery start --platforms instagram,tiktok --mode category
```

```bash
tiktok-miner discovery trending --topic fitness --platform instagram --limit 30
```

### Refresh Existing

Refresh data for existing creators:

```bash
tiktok-miner discovery refresh [identifier] [options]
```

#### Arguments

- `[identifier]` - Optional creator ID. If omitted, refreshes all

#### Options

- `--platform <platform>` - Refresh specific platform only
- `--priority <high|normal|low>` - Job priority (default: normal)
- `--stale-only` - Only refresh creators not updated recently
- `--older-than <hours>` - Refresh creators older than X hours

#### Examples

**Refresh a specific creator:**

```bash
tiktok-miner discovery refresh abc123 --priority high
```

**Refresh stale data:**

```bash
tiktok-miner discovery refresh --stale-only --older-than 48
```

**Refresh all YouTube data:**

```bash
tiktok-miner discovery refresh --platform youtube
```

### Generate Discovery Report

Generate a discovery performance report:

```bash
tiktok-miner discovery report [options]
```

#### Options

- `--start <date>` - Start date (ISO format or relative like "7 days ago")
- `--end <date>` - End date (default: now)
- `--format <table|json|html>` - Output format (default: table)
- `--output <path>` - Save to file

#### Examples

**Report for last 7 days:**

```bash
tiktok-miner discovery report --start "7 days ago"
```

**Generate HTML report:**

```bash
tiktok-miner discovery report \
  --start "2026-03-01" \
  --end "2026-03-06" \
  --format html \
```

## Discovery Configuration

### Quality Thresholds

The discovery system evaluates creators based on multiple factors:

```typescript
{
  qualityThreshold: 60,        // Minimum score to add (0-100)
  minFollowers: 1000,          // Minimum follower count
  maxFollowers: undefined,     // Maximum followers (undefined = no limit)
  minEngagementRate: 1.0,      // Minimum engagement rate (%)
  requiredPlatforms: [],       // Required platforms (empty = any)
  excludedCategories: [        // Categories to skip
    'adult',
    'controversial', 
    'spam'
  ],
  refreshIntervalHours: 24     // Hours between refreshes
}
```

### Quality Score Components

- **Follower Count** (20%) - Size of audience
- **Engagement Rate** (30%) - Audience interaction level
- **Content Frequency** (15%) - Posting consistency  
- **Audience Quality** (20%) - Fake follower detection
- **Growth Trend** (15%) - Recent growth trajectory

## Best Practices

### Regular Scheduled Discovery

Run discovery on a schedule for continuous growth:

```bash
# Cron job for daily trending discovery
0 10 * * * cd /app && bun cli/index.ts discovery trending --auto-add

# Weekly category exploration
0 0 * * 0 cd /app && bun cli/index.ts discovery category tech --limit 50
```

### Quality Over Quantity

Use higher quality thresholds to maintain database quality:

```bash
# High-quality creators only
tiktok-miner discovery start --quality-threshold 75 --min-followers 10000
```

### Monitor Discovery Metrics

Regularly check pipeline status and adjust settings:

```bash
# Check metrics
tiktok-miner discovery status

# Generate weekly reports
tiktok-miner discovery report --start "7 days ago" --output weekly-report.html
```

### Platform-Specific Discovery

Focus on platforms relevant to your needs:

```bash
# YouTube and Instagram only
tiktok-miner discovery start --platforms youtube,instagram
```

## Troubleshooting

### High Rejection Rate

If too many creators are rejected:

- Lower quality threshold: `--quality-threshold 50`
- Reduce minimum requirements: `--min-followers 500`
- Check excluded categories configuration

### Rate Limiting

If hitting API rate limits:

- Reduce batch size and concurrency
- Increase interval between runs
- Use platform-specific API keys
- Consider premium API tiers

### Duplicate Detection

If duplicates are being added:

- Check duplicate detector logs
- Verify platform identifiers are correct
- Review cross-platform matching logic

## Advanced Usage

### Custom Evaluation Criteria

Implement custom evaluation logic by modifying the configuration:

```bash
tiktok-miner discovery start \
  --quality-threshold 70 \
  --min-followers 5000 \
  --min-engagement 3.5
```

### Multi-Stage Discovery

Run discovery in stages for better control:

```bash
# Stage 1: Discover trending
tiktok-miner discovery trending --topic tech > discovered.json

# Stage 2: Review and filter
cat discovered.json | jq '.[] | select(.qualityScore > 75)' > approved.json

# Stage 3: Add approved creators
cat approved.json | while read creator; do
  tiktok-miner creators add $(echo $creator | jq -r '.username') \
    --platform $(echo $creator | jq -r '.platform')
done
```

### Integration with Other Tools

Export discovery data for external analysis:

```bash
# Export to data warehouse
tiktok-miner discovery report --format json --start "30 days ago" | \
  psql -h warehouse.example.com -d analytics -c "
    COPY discovery_metrics FROM STDIN WITH (FORMAT json)
  "
```
