## Overview

The `query-db` command provides powerful tools for querying and exploring data in the TikTok Miner PostgreSQL database. It supports both structured queries with convenient options and raw SQL for advanced use cases.

## Command Syntax

```bash
tiktok-miner query-db [sql] [options]
```

### Arguments

- `[sql]` - Optional raw SQL query to execute

### Options

- `-u, --username <username>` - Search for creators by username
- `-r, --repo <repo>` - Search repositories by URL or ID
- `-j, --jd <id>` - Search job descriptions by ID
- `-t, --tag <tag>` - Search job descriptions by tag
- `-l, --limit <number>` - Limit number of results
- `--repos` - Include owned repositories for user queries
- `--contributions` - Include contributed repositories for users
- `--contributors` - Include contributors for repository queries

## Database Schema

### Main Tables

#### CreatorProfile

Stores creator profile information:

```sql
CREATE TABLE CreatorProfile (
  id                       TEXT PRIMARY KEY,
  name                     TEXT NOT NULL,
  email                    TEXT,
  bio                      TEXT,
  profileImageUrl          TEXT,
  category                 TEXT,
  tags                     TEXT[],
  isVerified               BOOLEAN DEFAULT false,
  platformIdentifiers      JSONB,
  compositeEngagementScore FLOAT,
  totalReach               INTEGER DEFAULT 0,
  averageEngagementRate    FLOAT,
  contentFrequency         FLOAT,
  audienceQualityScore     FLOAT,
  lastSync                 TIMESTAMP DEFAULT NOW(),
  syncStatus               TEXT DEFAULT 'PENDING',
  createdAt                TIMESTAMP DEFAULT NOW(),
  updatedAt                TIMESTAMP DEFAULT NOW()
);
```

#### YoutubeMetrics

YouTube-specific metrics:

```sql
CREATE TABLE YoutubeMetrics (
  id                TEXT PRIMARY KEY,
  creatorProfileId  TEXT UNIQUE REFERENCES CreatorProfile(id),
  channelId         TEXT UNIQUE,
  subscriberCount   INTEGER DEFAULT 0,
  videoCount        INTEGER DEFAULT 0,
  viewCount         BIGINT DEFAULT 0,
  averageViews      INTEGER DEFAULT 0,
  averageLikes      INTEGER DEFAULT 0,
  averageComments   INTEGER DEFAULT 0,
  engagementRate    FLOAT DEFAULT 0,
  channelName       TEXT,
  channelUrl        TEXT,
  description       TEXT,
  lastSync          TIMESTAMP DEFAULT NOW()
);
```

#### InstagramMetrics

Instagram-specific metrics:

```sql
CREATE TABLE InstagramMetrics (
  id                TEXT PRIMARY KEY,
  creatorProfileId  TEXT UNIQUE REFERENCES CreatorProfile(id),
  username          TEXT UNIQUE,
  followerCount     INTEGER DEFAULT 0,
  followingCount    INTEGER DEFAULT 0,
  postCount         INTEGER DEFAULT 0,
  averageLikes      INTEGER DEFAULT 0,
  averageComments   INTEGER DEFAULT 0,
  engagementRate    FLOAT DEFAULT 0,
  lastSync          TIMESTAMP DEFAULT NOW()
);
```

#### TiktokMetrics

TikTok-specific metrics:

```sql
CREATE TABLE TiktokMetrics (
  id                TEXT PRIMARY KEY,
  creatorProfileId  TEXT UNIQUE REFERENCES CreatorProfile(id),
  username          TEXT UNIQUE,
  followerCount     INTEGER DEFAULT 0,
  followingCount    INTEGER DEFAULT 0,
  heartCount        BIGINT DEFAULT 0,
  videoCount        INTEGER DEFAULT 0,
  averageViews      INTEGER DEFAULT 0,
  averageLikes      INTEGER DEFAULT 0,
  engagementRate    FLOAT DEFAULT 0,
  lastSync          TIMESTAMP DEFAULT NOW()
);
```

#### TwitterMetrics

Twitter-specific metrics:

```sql
CREATE TABLE TwitterMetrics (
  id                TEXT PRIMARY KEY,
  creatorProfileId  TEXT UNIQUE REFERENCES CreatorProfile(id),
  username          TEXT UNIQUE,
  followerCount     INTEGER DEFAULT 0,
  followingCount    INTEGER DEFAULT 0,
  tweetCount        INTEGER DEFAULT 0,
  listedCount       INTEGER DEFAULT 0,
  averageLikes      INTEGER DEFAULT 0,
  averageRetweets   INTEGER DEFAULT 0,
  engagementRate    FLOAT DEFAULT 0,
  lastSync          TIMESTAMP DEFAULT NOW()
);
```

## Common Queries

### Find Top Performers

Find creators with highest engagement:

```bash
tiktok-miner query-db "SELECT name, category, totalReach, compositeEngagementScore FROM CreatorProfile WHERE compositeEngagementScore > 80 ORDER BY compositeEngagementScore DESC LIMIT 10"
```

### Category Analysis

Analyze creators by category:

```bash
tiktok-miner query-db "SELECT category, COUNT(*) as total, AVG(totalReach) as avg_reach, AVG(compositeEngagementScore) as avg_score FROM CreatorProfile GROUP BY category HAVING COUNT(*) > 10 ORDER BY avg_score DESC"
```

### Platform Comparison

Compare presence across platforms:

```bash
tiktok-miner query-db "SELECT COUNT(DISTINCT c.id) as total, COUNT(DISTINCT y.id) as youtube, COUNT(DISTINCT i.id) as instagram, COUNT(DISTINCT t.id) as tiktok, COUNT(DISTINCT tw.id) as twitter FROM CreatorProfile c LEFT JOIN YoutubeMetrics y ON c.id = y.creatorProfileId LEFT JOIN InstagramMetrics i ON c.id = i.creatorProfileId LEFT JOIN TiktokMetrics t ON c.id = t.creatorProfileId LEFT JOIN TwitterMetrics tw ON c.id = tw.creatorProfileId"
```

### Recently Added

Find recently added creators:

```bash
tiktok-miner query-db "SELECT name, category, totalReach, createdAt FROM CreatorProfile WHERE createdAt > NOW() - INTERVAL '7 days' ORDER BY createdAt DESC"
```

### Stale Data

Find creators needing sync:

```bash
tiktok-miner query-db "SELECT name, category, lastSync FROM CreatorProfile WHERE lastSync < NOW() - INTERVAL '48 hours' ORDER BY lastSync ASC LIMIT 20"
```

## Best Practices

### Use Parameterized Queries

When possible, use the structured options instead of raw SQL to avoid SQL injection:

```bash
# Good: Using options
tiktok-miner query-db --username "user input" --limit 10

# Risky: Raw SQL with user input (be careful!)
tiktok-miner query-db "SELECT * FROM CreatorProfile WHERE name = 'user input'"
```

### Limit Large Queries

Always use LIMIT for queries that might return many rows:

```bash
tiktok-miner query-db "SELECT * FROM CreatorProfile LIMIT 100"
```

### Use Indexes

The database has indexes on frequently queried columns:

- `category`
- `compositeEngagementScore`
- `totalReach`

Structure queries to use these indexes for better performance.

### Transaction Safety

The query-db command is read-only by default. For write operations, use the appropriate creator or discovery commands.

## Troubleshooting

### Connection Errors

If you see database connection errors:

```
Error: DATABASE_URL is not defined
```

Ensure your `.env` file contains:

```bash
DATABASE_URL=postgresql://user:password@host:5432/database
```

### Query Syntax Errors

For SQL syntax errors, check your query syntax:

```bash
# Error
tiktok-miner query-db "SELECT * FORM CreatorProfile"
# Error: relation "form" does not exist

# Correct
tiktok-miner query-db "SELECT * FROM CreatorProfile"
```

### Timeout Errors

For slow queries, add appropriate indexes or use LIMIT:

```bash
tiktok-miner query-db "SELECT * FROM CreatorProfile LIMIT 100"
```
